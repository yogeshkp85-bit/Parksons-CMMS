# Testing Strategy

The `tests/` directory has been scaffolded to support a robust test suite in future phases.

## Directory Layout
- `/tests/unit/`: Tests individual functions or service methods in isolation using mocks.
- `/tests/integration/`: Tests database queries (Prisma) and service-to-database interactions.
- `/tests/e2e/`: Tests full HTTP requests and responses (using Supertest) across the controller, service, and test database.
